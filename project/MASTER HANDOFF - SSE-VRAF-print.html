Visión del producto
Qué es el SSE-VRAF, objetivos y principios rectores
Qué es
Centro de Operaciones Digital
Sistema de gestión estratégica basado en Procesos Institucionales. No es un ERP ni un gestor documental. El usuario trabaja; el sistema genera indicadores, dashboards e informes automáticamente.
Principio rector
Capturar una sola vez
Cada dato se registra exactamente una vez. El sistema lo reutiliza en todos los módulos: procesos, KPIs, reportes e historial. Cero duplicidad de captura.
Usuarios
2 perfiles · 6 workspaces
Administrador General (Vicerrectora + Jefe RRHH): acceso transversal total.
Usuario Operativo: solo su workspace y sus asignaciones según RBAC.
Alcance
Arquitectura modular y escalable
VRAF · Recursos Humanos · Contabilidad · Compras · Mantenimiento · Salud y Seg. Ocup. Nuevos módulos se agregan sin reescribir el núcleo.
02
Arquitectura
Decisiones definitivas — no modificar sin acuerdo del Product Owner
Cadena obligatoria · toda entidad debe pertenecer a esta jerarquía o no existe
Plan→ Objetivo→ Proyecto→ Proceso ★→ Procedimiento→ Actividad→ Evidencia→ Indicador→ Dashboard→ Reporte
NÚCLEO
Proceso Institucional
Entidad central e inmutable. Todo el sistema orbita alrededor del Proceso.
DATO ÚNICO
Sin duplicación
Un dato se captura una vez. KPIs y reportes leen directamente de la BD.
MODULARIDAD
Plug-in architecture
Nuevos módulos se agregan sin modificar el núcleo del sistema.
AUTOMATIZACIÓN
Event-driven
Toda acción dispara eventos asíncronos BullMQ. El usuario trabaja; el sistema actualiza.
SEGURIDAD
RBAC por workspace
Permisos en DB. Middleware Next.js valida cada ruta. Operativo solo ve su unidad.
TRAZABILIDAD
Audit log inmutable
INSERT-only en HistorialAudit. Soft-delete en todas las entidades principales.

07
Reglas de negocio
10 reglas oficiales · sin duplicados · fuente única de verdad
R01
Sin datos duplicados
Cada dato se registra exactamente una vez. Indicadores y reportes leen de la BD directamente sin recaptura.
Fundamental
R02
Sin actividades aisladas
Toda actividad pertenece obligatoriamente a un Proceso. Bloquear creación de actividades sin procesoId en backend.
Arquitectura
R03
Sin procesos sin alineación estratégica
Todo proceso debe tener proyectoEstratégicoId. Bloquear en validación del backend (Zod).
Arquitectura
R04
Evidencia dispara automatización
Al insertar una Evidencia → recalcular KPI → actualizar dashboard → cerrar actividad si completa → registrar en HistorialAudit → notificar.
Automatización
R05
RBAC estricto por workspace
Middleware Next.js verifica wsId en cada petición. Operativo solo ve su unidad. Admin General ve todo.
Seguridad
R06
Audit log inmutable
INSERT en HistorialAudit es de solo escritura. No UPDATE, no DELETE. Soft-delete en entidades principales con campo deletedAt.
Trazabilidad
R07
Formularios como JSON Schema
La definición de formularios se almacena en Formulario.schema (jsonb). FormRenderer los interpreta dinámicamente sin código nuevo.
Escalabilidad
R08
SLA automático
Al crear un proceso: fechaLimite = fechaCreacion + sla_dias. BullMQ CRON genera alerta 48 h antes del vencimiento.
Automatización
R09
Herencia de plantilla
Al instanciar un proceso desde plantilla: hereda etapas, actividades, evidencias e indicadores del blueprint completo.
Escalabilidad
R10
Versiones de formulario
Publicar un formulario crea una nueva versión. Las versiones anteriores permanecen en BD. Soft-versioning con campo version.
Trazabilidad

Instrucción final para Claude Code
El prototipo es la fuente de verdad · implementar exactamente lo que se muestra
Implementa el SSE-VRAF con Next.js 14 App Router + TypeScript + Prisma + PostgreSQL
Cada pantalla del prototipo (9 archivos .dc.html) define el 100% del UX/UI. No redesignar. No agregar funcionalidades. Solo implementar. El modelo de datos está en el Data Studio. Los componentes del Form Builder se traducen a componentes React con JSON Schema. El Process Builder define la lógica del motor de procesos. Las automatizaciones (SI→ENTONCES) se implementan como BullMQ workers. El principio rector es: capturar una sola vez la información para reutilizarla en todo el sistema.
